FAQ & Troubleshooting

Common questions and fixes

Answers to everything that comes up. If something isn't here, check About Ogma or open the browser console for error details.

Data and storage

I cleared my browser cache and lost my pinned results. Can I recover them?

No — the data is gone if you cleared site data. Ogma stores everything locally in your browser. Clearing cache, cookies, or site data removes it permanently. There is no server backup.

To prevent this in future:

  • Open History (📋), click "Copy all as Markdown" before clearing anything
  • On Chrome: when clearing data, untick "Hosted app data" or "IndexedDB/localStorage" — clear just the HTTP cache
  • In private/incognito mode, data is wiped when the window closes. Don't use incognito for Ogma unless you're exporting before you close.
⚠ Data loss situations to avoid
  • Clearing site data or cookies in browser settings
  • Using private/incognito mode without exporting first
  • Switching browsers (data does not sync between Chrome and Firefox)
  • Switching devices (data stays on the original device)

Where exactly is my data stored?

IndexedDB stores: session state (last result, history ring, pinned cards, FP tracker state). Database name: fate_generator.

localStorage stores: theme, text size, help level, GM mode, universal merge preference, table customisations, PWA install dismissal, visit count.

To view it: open browser DevTools → Application → Storage → IndexedDB / Local Storage → (your site origin).

Can I transfer my data to another device or browser?

Not directly. The workaround: export pinned cards as Markdown from History, then paste them into a note or document on the new device. Ogma doesn't have an import-from-backup feature; what you export is the content, not the app state.

Offline and installation

Does Ogma work offline?

Yes — fully. After your first visit, the service worker caches all 740KB of the app. You can open the app with no internet, run sessions, pin results, export Markdown — everything works.

The only thing that requires internet: the initial first load to install the service worker, and future update checks (which happen silently in the background).

I see "Update available — reload?" — should I click it?

Yes. This means a new version of Ogma has been deployed. Your data is not affected by updates. Clicking reload fetches the latest files and clears the old service worker cache. If you don't click it, the old version continues to work until you next close the tab.

The app isn't loading updates — it's stuck on an old version.

Force-refresh bypasses the service worker: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac). Alternatively: DevTools → Application → Service Workers → click "Skip waiting" next to the waiting worker.

Projector Mode

The projector screen shows "Waiting for GM to push a result" but nothing happens when I press V.

Check these in order:

  1. Both tabs must be in the same browser — BroadcastChannel only works within the same browser on the same device. Chrome and Firefox can't communicate.
  2. The projector page must be open — open projector.html in a new tab first, then push from the campaign page.
  3. In file:// mode — BroadcastChannel may be unavailable. The localStorage polling fallback activates automatically, but there's a ~500ms delay. This is expected behaviour.

Can I use the projector on a different device (e.g. Chromecast)?

Not natively — BroadcastChannel and localStorage only work within the same browser on the same machine. For a different device, you'd need to run Ogma on a web server and open both pages there. The file:// mode doesn't support cross-device sync.

Common UI questions

What does ↺ Reset do in the FP Tracker?

It resets every PC's current Fate Points to their individual Refresh value. Use this at the start of each session. It does not affect pinned cards, history, or any other app state.

The sidebar is showing a "Sessions" tab I haven't seen before.

The sidebar has two tabs: Generate (the 16 generators grouped by type) and Session (tools for live play: FP Tracker, History, Projector link, Full Session, and navigation). Click the tab labels at the top of the sidebar to switch.

How do I get back to the world list?

Click OGMA in the topbar (the wordmark) or the Home crumb in the breadcrumb. This returns you to the landing page where all 7 campaign worlds are listed.

I'm getting an error in the browser console — what do I do?

Open DevTools (F12), go to the Console tab, and look for red errors. The most common ones:

  • LS is not defined — db.js failed to load. Usually caused by a script order issue in a custom deployment. Ensure db.js loads before ui.js.
  • CAMPAIGNS is not defined — the campaign data file (e.g. data/fantasy.js) failed to load. Check the network tab for 404 errors.
  • React is not defined — the CDN failed. Check your internet connection or use the offline ZIP.

Can I contribute content or report bugs?

Yes — the project is on GitHub: github.com/brs165/ogma-fate. Open an issue for bugs, or a pull request for content additions. Table data lives in data/[world].js files.
